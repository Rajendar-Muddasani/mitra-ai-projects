AI Resume Screener & Interview Copilot

Build a tool that takes a job description and a batch of resumes, extracts key information, scores each resume against the job requirements using an LLM, and generates tailored interview questions.

what you build

An HR tech tool that ranks resumes and generates personalised interview questions — demonstrating NLP, LLM APIs, and Streamlit.

Core Features

  • Upload multiple resumes (PDF/DOCX) and a job description
  • Extract text using PyMuPDF and parse key sections (skills, experience, education)
  • Score each resume 0–100 using LLM with structured prompt
  • Ranked leaderboard showing top candidates with scores and summaries
  • Generate 5 personalised interview questions per candidate
  • Export results to CSV for HR team

  • Familiarity with Streamlit for UI (or willingness to learn)

code patterns

Key Code Patterns

text_extraction.py 
import fitz  # PyMuPDF

def extract_resume_text(pdf_path: str) -> str:
    doc = fitz.open(pdf_path)
    text = ""
    for page in doc:
        text += page.get_text()
    return text.strip()
scoring_prompt.py 
def score_resume(resume_text, job_desc, client):
    prompt = f"Score this resume 0-100.\nJOB: {job_desc}\nRESUME: {resume_text}"
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role":"user","content":prompt}],
        response_format={"type":"json_object"}
    )
    return json.loads(response.choices[0].message.content)

viva prep

20 Viva Questions with Answers

Q1. What is the difference between keyword matching and LLM-based resume scoring?
Keyword matching checks for exact word presence — it misses synonyms and context. LLM scoring understands meaning: "managed a team" matches "leadership experience" even without the exact words.
Q2. How does your scoring system avoid bias?
The prompt instructs the LLM to focus only on skills and experience relevant to the job description, explicitly excluding age, gender, and personal details. The structured output enforces objective criteria.
Q3. Why did you use PyMuPDF for PDF extraction instead of PyPDF2?
PyMuPDF (fitz) provides better layout preservation and handles complex PDFs with tables, images, and multi-column layouts more reliably than PyPDF2.
Q4. What is the response_format: json_object parameter in OpenAI API?
It forces the model to return valid JSON, preventing parsing errors from extra text or markdown code blocks around the JSON output.
Q5. How would you handle rate limiting from the OpenAI API?
Implement exponential backoff: catch RateLimitError, wait 2^n seconds, retry up to 3 times. For batch processing, add time.sleep(1) between requests.
Q6. What are the limitations of your resume screener?
1) Only works with text-based PDFs, not scanned images. 2) LLM costs scale with volume. 3) AI screening can have biases from training data. 4) No support for non-English resumes.
Q7. How would you scale this to process 10,000 resumes per day?
Use async/await with asyncio for parallel API calls. Use a task queue (Celery + Redis). Consider OpenAI Batch API (50% cheaper for non-real-time). Cache embeddings for repeated job descriptions.
Q8. What is Streamlit and why did you choose it over Flask/FastAPI?
Streamlit converts Python scripts into interactive web apps with minimal code. It handles state, file uploads, and UI components automatically. Flask/FastAPI require HTML/CSS/JS knowledge. For rapid prototyping, Streamlit is faster.
Q9. How do you validate the LLM output JSON structure?
Use Pydantic models to validate: define a ResumeScore model with required fields and types. Catch ValidationError and retry with a clarified prompt if the structure is wrong.
Q10. What metrics would you use to evaluate the screener quality?
Compare AI scores against HR expert rankings on a test set. Measure precision@k (are the top-k AI-selected candidates actually good?). Track consistency: same resume scored twice should get similar scores.
Q11. Explain how interview question generation works in your system.
After scoring, a second prompt receives the resume text, job description, and identified skill gaps. It generates 5 specific questions targeting the candidate's gaps or clarifying their strongest experiences.
Q12. Why is it important to show the matched skills and gaps, not just the score?
A score alone is a black box — HR teams cannot justify why a candidate was ranked low. Showing matched skills and gaps makes the system explainable and helps recruiters make informed decisions.
Q13. How would you add support for scanned PDF resumes?
Use AWS Textract or Google Cloud Vision OCR to extract text from scanned images before passing to the LLM pipeline. This adds latency and cost but handles image-based PDFs.
Q14. What is the difference between extracting text and understanding a resume?
Extraction gives raw text. Understanding means identifying sections (skills, experience, education), parsing dates, normalising job titles, and mapping to standardised skills taxonomies.
Q15. How do you handle resumes in different formats (2-page vs 1-page, functional vs chronological)?
The LLM handles format variation naturally since it understands language context. For structure, the extraction focuses on full text rather than trying to parse sections programmatically.
Q16. What is the approximate cost per resume screening with gpt-4o-mini?
A resume is ~500 tokens. Job description ~200 tokens. Response ~300 tokens. Total ~1000 tokens. At $0.15/1M input + $0.60/1M output: approximately $0.0003 per resume — very cheap for automated screening.
Q17. How would you build a feedback loop to improve the system over time?
Collect HR feedback on AI rankings (thumbs up/down per candidate). Store feedback in a database. Periodically fine-tune the scoring prompt or a fine-tuned model on the collected feedback data.
Q18. What data protection considerations apply to resume screening?
Resumes contain personal data (PII). Under GDPR/PDPA: inform candidates that AI is used, don't store resumes longer than necessary, implement data deletion, avoid sending PII to third-party APIs if possible (use local models).
Q19. How would you explain this project in a placement interview?
I built an automated recruitment tool that uses OpenAI's API to score resumes against job descriptions, rank candidates, and generate interview questions. It handles PDF extraction, structured LLM outputs, and Streamlit UI.
Q20. What would you add next if you had 2 more weeks?
ATS integration via email parsing, multi-language support using translation before scoring, a candidate notification system, and a similarity search to find historical candidates matching new jobs.
