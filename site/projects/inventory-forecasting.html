what you build

A real-world inventory management tool that businesses use daily — combining statistical forecasting, data visualisation, and interactive UI.

Core Features

  • CSV upload with automatic column detection (date, quantity)
  • Exploratory data analysis: trend, seasonality, and outlier detection
  • ARIMA model with ADF stationarity test and auto-order selection
  • Facebook Prophet model with holiday effects support
  • Model comparison: MAPE, RMSE, MAE metrics side-by-side
  • Interactive plotly charts with confidence intervals
  • Forecast download as CSV for supply chain planning

code patterns

Key Code Patterns

stationarity_test.py 
from statsmodels.tsa.stattools import adfuller

def test_stationarity(series) -> dict:
    result = adfuller(series.dropna())
    return {
        'adf_stat':  result[0],
        'p_value':   result[1],
        'is_stationary': result[1] < 0.05,  # p < 0.05 = stationary
        'critical_values': result[4]
    }
arima_forecast.py 
from statsmodels.tsa.arima.model import ARIMA
import pmdarima as pm  # auto_arima

# Auto-select best (p, d, q)
auto = pm.auto_arima(train, seasonal=False, stepwise=True, error_action='ignore')
model = ARIMA(train, order=auto.order).fit()

# Forecast next 30 periods with confidence intervals
forecast = model.get_forecast(steps=30)
fc_mean  = forecast.predicted_mean
fc_ci    = forecast.conf_int(alpha=0.05)

from prophet import Prophet

viva prep

20 Viva Questions with Answers

Q1. What is time series forecasting and why is it important for inventory management?
Time series forecasting predicts future values based on historical sequential data. For inventory, it prevents stockouts (running out of goods) and overstock (tying up capital in unsold goods).
Q2. What is stationarity and why does ARIMA require it?
A stationary series has constant mean, variance, and autocorrelation over time — no trend or changing seasonality. ARIMA assumes stationarity to make valid statistical predictions.
Q3. What is the Augmented Dickey-Fuller test and how do you interpret it?
The ADF test checks for a unit root (non-stationarity). p-value < 0.05: reject null hypothesis, series is stationary. p-value ≥ 0.05: series is non-stationary, apply differencing.
Q4. Explain the (p, d, q) parameters in ARIMA.
p = AR order (how many past values to use), d = degree of differencing (how many times to difference to achieve stationarity), q = MA order (how many past forecast errors to use).
Q5. What is differencing and when do you apply it?
Differencing: y'(t) = y(t) - y(t-1). Removes trend. Second differencing removes quadratic trend. Apply when ADF test shows non-stationarity (p ≥ 0.05).
Q6. How does Facebook Prophet handle seasonality differently from ARIMA?
Prophet uses Fourier series to model seasonality and allows multiple seasonalities (yearly, weekly, daily) simultaneously. ARIMA uses ACF/PACF-based lag terms. Prophet is more flexible for irregular/multiple seasonalities.
Q7. What is MAPE and when is it preferred over RMSE?
MAPE = Mean Absolute Percentage Error. Scale-free (percentage), so you can compare models across series with different magnitudes. RMSE is better when large errors are disproportionately costly.
Q8. How do confidence intervals work in your forecast?
The 95% CI means the model predicts the true value will fall within the band 95% of the time. Wider bands = more uncertainty (further into the future or high variance data).
Q9. What is seasonal decomposition and what does it show?
STL or classical decomposition separates a series into Trend + Seasonality + Residual components. It reveals the underlying structure and helps choose the right model.
Q10. How would you handle missing values in the time series?
Options: forward fill (use last known value), backward fill, linear interpolation, or seasonal interpolation. The choice depends on what caused the gap (weekend, holiday, data error).
Q11. What would cause ARIMA to perform poorly on your data?
Structural breaks (sudden policy change, COVID effect), multiple seasonalities, non-linear relationships, very short history, or extreme outliers. Prophet handles these better.
Q12. How do you evaluate forecasting models fairly?
Time series cross-validation (expanding window): train on t1..t_n, test on t_{n+1}..t_{n+h}, roll forward. Never use future data for training — this causes data leakage.
Q13. What is the changepoint_prior_scale parameter in Prophet?
Controls how flexible the trend is. Higher value = more changepoints detected (risk of overfitting). Lower = smoother, more regularised trend. Default 0.05 is a good starting point.
Q14. How would you add holiday effects to your forecast?
Prophet: create a DataFrame with holiday name, date, and lower/upper window (days before/after). Pass to Prophet(holidays=holidays_df). This adds indicator variables for holiday periods.
Q15. What is auto_arima and how does it work?
pmdarima.auto_arima() uses stepwise search (AIC minimisation) to find the best (p,d,q) automatically. It tests combinations of orders, fits ARIMA, and returns the model with the lowest AIC.
Q16. How would you deploy this to handle 100 users simultaneously?
Streamlit runs single-threaded by default. Use Streamlit Cloud or Docker + Kubernetes for multi-user. Add caching (@st.cache_data) on data loading. Store user sessions in Redis.
Q17. What is the difference between in-sample and out-of-sample forecasting?
In-sample: forecast within the training period (checks model fit but not generalisation). Out-of-sample: forecast beyond the training period (true test of forecasting ability).
Q18. How would you handle a product with no historical data?
Cold start problem. Options: use similar products (cross-sectional forecast), use global model trained on all products, or apply a simple baseline (category average).
Q19. Why did you choose Streamlit over a custom web frontend?
Streamlit allows rapid prototyping in pure Python — no HTML/CSS/JS required. It handles state, file uploads, and reactive updates automatically. Ideal for data science demos and dashboards.
Q20. What extension ideas could make this a more complete product?
Multi-product forecasting (one model per SKU), reorder point calculation (when to order based on forecast + safety stock), supplier lead time integration, and email alerts when stock falls below threshold.
