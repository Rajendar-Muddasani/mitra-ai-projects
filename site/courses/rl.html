What is Reinforcement Learning?

An agent learns to take actions in an environment by receiving reward signals. Unlike supervised learning, there are no labeled examples — the agent discovers the best strategy by trial and error.

Core Components

ComponentDescription
AgentThe learner/decision maker
EnvironmentEverything the agent interacts with
State (s)The current situation of the agent
Action (a)What the agent does in a state
Reward (r)Feedback signal — positive for good actions, negative for bad
Policy (π)The agent's strategy: state → action mapping
Value FunctionExpected future rewards from a state following a policy
Markov Decision ProcessThe mathematical framework for RL problems

RL vs. Supervised vs. Unsupervised

Supervised

Labeled (x, y) pairs. Learn to predict y from x. Teacher provides correct answers.

Unsupervised

No labels. Find structure in data. Clustering, dimensionality reduction.

Reinforcement

No labels — just reward signals. Learn by interacting with an environment. Sequential decisions.

Q-Learning

Model-free RL for discrete state spaces. Learn a Q-table that maps (state, action) pairs to expected future rewards. The agent picks the action with the highest Q-value.

Bellman Equation

Q(s, a) ← Q(s, a) + α [r + γ · max Q(s', a') − Q(s, a)]

α = learning rate · γ = discount factor (importance of future rewards)

Key Concepts

ConceptMeaning
Exploration vs. ExploitationTry new actions (explore) vs. use best known action (exploit)
ε-greedyWith probability ε, take a random action; else take the best action
Discount factor γ0 = myopic (care only about immediate reward); 1 = far-sighted

Deep Q-Networks (DQN)

Replace the Q-table with a neural network when the state space is too large for a table (e.g., pixel images in Atari games). Two key innovations: experience replay and target networks.

DQN Innovations

InnovationWhy it helps
Experience ReplayStore (s, a, r, s') transitions in a buffer; sample mini-batches randomly — breaks correlation
Target NetworkA separate slowly-updated copy of the Q-network — stabilizes training by providing a fixed target
CNN for pixelsConvNet extracts features from raw pixel observations

Policy Gradients

Directly optimize the policy (the action-selection function) instead of learning a value function. REINFORCE is the baseline algorithm; Actor-Critic combines both approaches.

Value-Based vs. Policy-Based

Value-Based (Q-learning, DQN)
  • Learn Q(s,a), then derive policy
  • Works well for discrete actions
Policy-Based (PG, PPO)
  • Directly parameterize π(a|s)
  • Works for continuous action spaces

PPO — Proximal Policy Optimization

The most widely used modern RL algorithm. PPO is stable, scalable, and easy to tune. It's used by OpenAI for RLHF (training ChatGPT) and by DeepMind for game-playing agents.

PPO Key Ideas

IdeaWhat it does
Clipping objectiveLimits how much the policy can change in a single update — prevents instability
Multiple update epochsReuse each batch of experience for K epochs — more sample efficient
Advantage estimation (GAE)Better estimate of "how much better than average was this action?"
Entropy bonusEncourages exploration by rewarding uncertainty in the policy

Real-World Reinforcement Learning

RL beyond games. Real-world applications have sparse rewards, safety constraints, and costly exploration — requiring different approaches than simulated environments.

Application Areas

DomainRL ApplicationKey challenges
Recommendation SystemsChoose which item to show next to maximize engagementExploration costly, delayed reward
Supply ChainOptimize inventory levels and order quantitiesNoisy demand, lead time uncertainty
Traffic ControlOptimize signal timing to reduce congestionMulti-agent, non-stationary environment
RLHF (LLMs)Fine-tune language models to human preferencesReward hacking, alignment
RoboticsTeach robots to walk, grasp, and navigateSim-to-real gap, safety
